## 1. 業務背景（一定要先懂）

- 業主做**不動產抵押借款**（二胎、民間借貸、代書案件）。
- 「案源」= 一個可能借款的客戶 + 他名下的不動產（房子）。
- 每天會從各種管道拿到名單（公司總檔、Excel、謄本），要判斷這個案子值不值得做、能借多少。
- 核心動作：**評估不動產價值 → 算可貸金額 → 追蹤客戶 → 簽約**。
- 「謄本」= 地政機關的不動產登記資料（標示部/所有權部/他項權利部），是判斷案子的關鍵文件。

### 借款相關名詞
- **他項權利部** = 房子上已經有的抵押（銀行貸款、民間借貸），決定還能再借多少。
- **預告登記** = 房子被某人卡住不能移轉，通常是民間金主放款後設的，是重要風險警訊。
- **土增稅、土地使用分區** = 評估土地價值用。
- 金額單位通常用「**萬**」。利率民間用「**分**」（月利率，1分=1%/月）。

## 4. 架構

### 追蹤系統（index.html）
- **Firebase Authentication**（email/密碼）登入。
- **Firestore** 是主要儲存，每個使用者一份 `users/{uid}` document。
- **localStorage**（key = `loan_crm_fb_v1`）當本地快取，靠 `uid` 隔離不同帳號。
- **載入策略（local-first）**：登入 → `loadLocal()` 先秒顯快取 → `render()` → 背景 `syncFromFirestoreIfNewer()` 比時間戳，雲端較新才覆蓋。
- Firebase 設定：projectId = `loan-crm-a60e0`（設定在 index.html 約 317 行）。

### Firestore 資料結構 `users/{uid}`
```
{
  data:       JSON字串 {clients:[...], nextId},  // 主資料
  backup:     JSON字串 {clients, nextId},        // 備份（只在非空時更新）
  backupAt:   時間戳,
  gasUrl:     共用同步 GAS 網址,
  companyUrl: 個人總檔 GAS 網址,
  updatedAt:  時間戳,
  sessionToken: 單一裝置登入控制
}
```

### 兩個 GAS（Google Apps Script）端點
1. **共用同步 GAS**（`GAS_URL`，寫死在 index.html 約 1111 行）— 舊的 Google Sheets 同步，全公司共用。
2. **個人總檔 GAS**（`COMPANY_URL`，每個使用者在 ⚙️ 設定自己填）— 讀取「自己的」Google Sheet 總檔 + 在「自己的」Google Drive 建客戶資料夾。
   - token 驗證用 `API_TOKEN = 'stanford87'`。
   - 個人 GAS 的完整程式碼見第 7 節。
   - **必須用 V8 執行階段**（不是舊的 Rhino），且要授權 Drive 完整權限（見坑 8.4）。

### 整理工具（整理工具.html）
- 純前端，無登入、無雲端。用 SheetJS 讀 Excel。
- Union-Find 演算法做去重（同電話/身分證/建號/地址視為同一人）。
- 輸出整理後的 Excel。

## 5. 分類系統（heat）

客戶分 6 類，**順序固定**：

| 分類 | 說明 |
|---|---|
| `待分類` | 新進來還沒判斷的 |
| `覆訪` | 要再聯絡的 |
| `追很緊` | 熱門積極追蹤（原本叫「最新資料」，已改名）|
| `新鑫` | 新鑫融資的案子 |
| `已簽約` | 成交 |
| `無效案件` | 斷聯/塗銷/無效 |

- 程式內對應在 `hOrd` / `hCls`（約 2520 行）。
- 歷史包袱：舊資料可能有 `最新資料`，`migrateHeat()` 會自動轉成 `追很緊`。
- 總檔分頁名稱 → 分類的對應在 `sheetToHeat()`。整理工具的對應在 `guessCat()`。兩邊規則要一致。

## 6. 主要功能與對應函式（index.html）

| 功能 | 入口/函式 | 說明 |
|---|---|---|
| 從總檔更新 | `syncFromCompany()` → `processCompanyData()` | 抓個人 GAS 的 Sheet 資料，依分頁分類匯入 |
| XLS 匯入 | `openImport()` → `doImport()` | 上傳 Excel |
| 謄本匯入 | `openTenModal()` → 解析 PDF → `applyTenParsed()` | 用 PDF.js 解析地政謄本，自動填欄位 |
| **AI 匯入** | `openAiImport()` → `parseAiImport()` → `doAiImport()` | **見第 9 節，這是給 AI 整理資料用的固定格式入口** |

### 客戶資料結構（clients[] 的一筆）
重點欄位：
```
{
  id, name, phone, idno, heat, note, job,
```

## 7. 個人總檔 GAS 完整程式碼

業主和同事各自部署一份，貼到自己的 Google Sheet 綁定的 GAS 專案。
- 改 `SPREADSHEET_ID` 為自己的試算表 ID。
- 部署：**網頁應用程式**，執行身分=**我自己**，存取=**所有人**。
- 第一次要執行 `authorize` 函式並授權 Drive（完整權限，見坑 8.4）。

```javascript
var SPREADSHEET_ID = '試算表ID';
var TOKEN = 'stanford87';

function doGet(e) {
  var token = e.parameter.token || '', action = e.parameter.action || '';
  var callback = e.parameter.callback || 'callback';
  if (token !== TOKEN) return jsonp(callback, {status:'error', message:'invalid token'});

  if (action === 'createFolder') {
    var name = e.parameter.name || '未命名客戶';
    try {
      var parentName = '客戶資料';           // 客戶資料夾建在這個母資料夾底下
      var iter = DriveApp.getFoldersByName(parentName), parent;
      parent = iter.hasNext() ? iter.next() : DriveApp.createFolder(parentName);
      var folder = parent.createFolder(name);
      return jsonp(callback, {status:'ok', url: folder.getUrl()});
    } catch(err) { return jsonp(callback, {status:'error', message: err.toString()}); }
  }

  try {
    return ContentService.createTextOutput(callback + '(' + JSON.stringify(getSheetData()) + ')')
      .setMimeType(ContentService.MimeType.JAVASCRIPT);
  } catch(err) { return jsonp(callback, {status:'error', message: err.toString()}); }
}

function getSheetData() {
  var ss = SpreadsheetApp.openById(SPREADSHEET_ID), sheets = ss.getSheets();
  var SKIP = ['設定','config','Config','說明','Sheet1'], rows = [];
  rows.push(['電話姓名','拜訪過程','備註','建物編號','門牌地址','面積','建物所有權部','','','','','','','他項權利部','分頁']);
  rows.push(['電話/姓名欄','拜訪過程','備註','建物所有權編號','門牌地址','面積坪數','建物所有權部','','','','','','','他項權利部','分頁名稱']);
  for (var i=0;i<sheets.length;i++){
    var sh=sheets[i], nm=sh.getName(), skip=false;
    for (var s=0;s<SKIP.length;s++) if(SKIP[s]===nm){skip=true;break;}
    if(skip) continue;
    var data=sh.getDataRange().getValues();
    if(data.length<2) continue;
    for (var r=1;r<data.length;r++){
      var row=data[r], has=false;
      for (var c=0;c<row.length;c++) if(String(row[c]).trim()!==''){has=true;break;}
      if(!has) continue;
      var pad=[]; for(var c=0;c<14;c++) pad.push(c<row.length?row[c]:'');
      pad.push(nm); rows.push(pad);
    }
  }
  return rows;
}

function jsonp(cb,obj){
  return ContentService.createTextOutput(cb+'('+JSON.stringify(obj)+')')
    .setMimeType(ContentService.MimeType.JAVASCRIPT);
}

function authorize(){ DriveApp.getRootFolder(); SpreadsheetApp.openById(SPREADSHEET_ID); }
```

## 8. 踩過的坑（別重犯）

1. **跨帳號資料外洩**：換帳號時舊帳號資料被帶進來。根因有三層 — 舊快取沒 uid、登入/登出沒清空記憶體 `clients`、登入時自動 `syncData()` 抓到舊 GAS。解法：`loadLocal()` 嚴格比對 uid、登入登出都 `clients=[]; nextId=1;`、**移除登入自動同步**。改動這塊要非常小心。
2. **每次登入都跳「已從雲端同步」**：`saveLocal()` 和 `saveToFirestore()` 各自產生時間戳差幾毫秒。解法：同一個 `ts` 傳下去。
3. **跨裝置同步變空白**：`loadLocal()` 失敗→clients 空，但時間戳相同→不更新→空白。解法：`if (clients.length > 0 && firestoreTs <= localTs) return false;`（clients 空時強制從雲端載）。
4. **GAS Drive 權限**：部署若「執行身分」選成「使用者」會沒權限；且若 `authorize` 只呼叫 `getFoldersByName`（讀），GAS 只給 `drive.readonly`，`createFolder`（寫）就失敗。解法：appsscript.json 明確宣告 `oauthScopes: ["https://www.googleapis.com/auth/drive", ".../spreadsheets"]`，先撤銷舊授權再重授權，部署執行身分=我自己。
5. **GAS 語法**：舊 Rhino 執行階段不支援箭頭函式/展開運算子/`?.`。要嘛用 V8，要嘛全用 `var`+`for`+`.concat()`。
6. **重複偵測誤判**：曾用「同姓+同字數」判同名→所有林X都被合併。解法：只用電話/身分證/建號/地址精準比對，移除姓氏猜測。

## 9. AI 匯入格式（重要 — 這是 AI 協作的核心）

業主會把**謄本或案源評估表**丟給 AI（就是你），你整理成**固定 JSON**，他貼進 app「🤖 AI 匯入」一鍵建立。因為欄位跟系統 1:1 對應，匯入不需模糊比對。

### JSON 格式
```json
[{
  "name": "姓名",
  "phone": "電話",
  "idno": "身分證",
  "heat": "待分類",
  "loanPurpose": "借款用途",
  "note": "備註（預告登記、抵押權人等警訊寫這）",
  "properties": [{
    "addr": "門牌地址", "purpose": "地段(如 忠福段)",
    "buildNo": "建號", "landNo": "地號",
    "mainArea": 主建坪, "subArea": 附屬坪, "commonArea": 公設坪, "area": 總坪,
    "bank": 銀行萬, "legal": 法人/新鑫萬, "private": 民間萬
  }],
  "eval": {
    "district":"區", "section":"地段", "landno":"地號", "landzone":"使用分區",
    "buildno":"建號", "addr":"地址", "community":"社區名稱",
    "buildarea":"建物坪", "parkval":"車位坪", "landarea":"土地坪",
    "floors":"總樓層", "age":"屋齡", "taxGeneral":"土增稅一般", "taxSelf":"土增稅自用",
    "condition":"屋況", "resistance":"抗性", "jinzhai":"凶宅查詢",
    "unitprice":"行情萬/坪", "rate":"利率分", "need":"客戶需求萬",
    "job":"職業收入", "evalNote":"總價/其他備註"
  }
}]
```
- `eval` 整包選填，key 與 `evalData` 完全一致，匯入時直接帶進「評估分頁」。
- 匯入會用 **身分證 > 電話** 精準比對既有客戶：對到=更新（備註附加、不覆蓋既有）、沒對到=新增。
- 預覽可逐筆勾選、**自選分類（下拉）**、把「更新」改成「新增」。

### 建號／地號的書寫慣例（業主習慣，務必遵守）
母號-子號，**子號去掉開頭的 0；子號全 0 就只留母號**：
| 謄本原文 | 寫成 |
|---|---|
| `3204-000` | `3204` |
| `3145-0000` | `3145` |
| `3204-0001` | `3204-1` |
| `3204-0012` | `3204-12` |
地段名（如「忠福段」）放 `purpose`/`eval.section`，建號地號欄只放純號碼。

### 解讀謄本的要點
- **所有權人姓名常被遮罩成「郭＊＊」**，但「預告登記義務人」「相關他項權利」欄位常露出全名，可互相補全。
- 身分證遮罩（`H223*****6`）就留空，別亂猜。
- 坪數：主建物(層次面積)、附屬建物、共有部分(公設) 分開抓，平方公尺 ×0.3025 = 坪。
- 屋齡 = 今年 - 建築完成民國年(+1911)。
- 他項權利部每筆抵押：銀行→`bank`、租賃/融資/新鑫等法人→`legal`、私人/不動產顧問公司→`private`。金額元÷10000=萬。
- **預告登記一定要寫進 note 當警訊**，這是業務判斷風險的關鍵。
